使用 Obsidian 搭配 hugo 与阿里 oss 搭建博客系统

2022-06-09 · 3 min · 473 words · Ivan | Edit

没想到刚用上 hexo，还不到 1 天就换成 hugo 了。看项目星星数量，hugo 显然是比 hexo 要多上不少，且主题的素质明显高出很多，猜测大概率也是颜值的助益。

安装#

  • 通过 brew install hugo 直接安装二进制文件
  • 另外如果没装 Git 一定要装上

初始化#

  • 通过命令 hugo new site 目录名称 来创建一个项目目录
  • 然后使用 git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1 命令克隆一个主题到 themes 子目录里
  • 修改项目中的 hugo.toml 配置文件中的 theme 参数为 PaperMod 这个主题
  • 最后在项目目录里使用 hugo server 就能启用一个本地服务器，看到站点预览了

关于 PaperMod 主题#

后来才知道，hugo 的配置文件有两种模式，默认是 hugo.toml 这个配置文件，而 PaperMod 主题需要使用 yml 格式的配置文件，如果站点已经创建，就删除掉 hugo.toml 配置文件，然后新建一个 config.yml 文件在根目录中，再填入主题官方给的默认配置内容：

写作#

  • 要新建一个文章则使用 hugo new posts/123.md 命令
  • 不过这显然不符合使用习惯，所以在 content 目录下创建一个硬链接，指向 Obsidian 的一个子目录，在该目录下直接通过 Obsidian 模板创建文档即可
  • 完成后使用 hugo 命令即可直接生成构建

部署#

利用阿里云 OSS 自带的 ossutil 工具直接上传到 oss，使用 Alfred 制作一个 workflow 依次执行以下命令：

cd /hugo目录
hugo
ossutil rm oss://bucket/blog -rf
ossutil cp public/ oss://bucket/blog/ -f --recursive

把以上四句命令集成在 Alfred 的 blog 命令下，即可直接一键生成、发布。

遇到的一些问题#

  • 在配置首页的描述文字时无法换行，配置文件对应的是 homeInfoParams 下的 Content ，后来发现在段落结尾打上 3 个空格，然后加上一个 \n 可以有效实现换行，同时使用 md 的语法也可以正常渲染。
  • 首页的文章排序一开始是乱的，发现是因为在 md 文档中头部的 date 参数格式配置不正确，后在配置文件的 params 下 DateFormat 参数修改为一致的格式即可，但是要注意必须使用 2006-01-02 这个年月日，随意使用别的日期也会无法识别。
